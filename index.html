HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML adds structure to the content of a webpage.

CSS

  • Inline CSS: Inline styles apply to a specific HTML tag (an element directly in the HTML file).
  • internal style sheet: Contains CSS rules for the webpage in the head section of the HTML file (a single change to the CSS rule will apply to all tagged page elements).
  • Inline CSS: Inline styles apply to a specific HTML tag (an element directly in the HTML file).
  • External CSS style sheet:the most common way to style a webpage, this is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier
  • Margin property:space we want around the outside of an element, and Padding property: adds space around the content inside an element.
  • Debug code example:"link rel="stylesheet" href="./assets/style.css"

GIT

  • Git status: checks what branch we are currently on
  • Git checkout -b branch-name: creates a new branch and switches to it
  • Git main branch command: git checkout main
  • Git new branch command: git checkout -b feature/add-html(name of the branch)
  • Stage your changes: git add -A
  • Commit the changes: git commit -m "added code to HTML file"
  • Pull in the base branch: git pull origin main
  • Push up all our changes:git push origin feature/add-html (name of the branch)

JavaScript

  • JavaScript is a programming language that allows users to interact with websites. One of its core features is variables, which store data types as values.
  • Variable is simply a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script.
  • Function is simply a powerful core component and a set of instructions that tells the computer how to perform a certain task.
  • Common JavaScript error: "Uncaught ReferenceError: topic is not defined"(check names of variables)
